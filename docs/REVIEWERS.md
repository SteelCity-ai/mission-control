# PR Review Assignments

## Required Reviewers
| Change type | Reviewer | Reason |
|-------------|----------|--------|
| Any code | **Han (QA)** | Quality gate — all code must pass QA review |
| Design/UX | **Leia (Design)** | Visual changes need design approval |
| Architecture | **Akbar (Architect)** | Structural changes need architecture review |
| Config/deploy | **Yoda (Router)** | Infrastructure changes need approval |

## Process
1. Open PR with description
2. Auto-assign Han for all PRs
3. If design changes → also assign Leia
4. If architecture changes → also assign Akbar
5. Han approves → merge
